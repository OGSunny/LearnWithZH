Building Buttons & GUIs

GUIs aren't just pretty—they're your game's front door. Let's make buttons that click, hover, and feel alive.

Starting a Basic GUI

Drop a ScreenGui into StarterGui. Inside? A Frame, then a TextButton. Boom, interactive.

LocalScript for Button Click

local Players = game:GetService("Players")
local player = Players.LocalPlayer
local playerGui = player:WaitForChild("PlayerGui")

local screenGui = Instance.new("ScreenGui")
screenGui.Parent = playerGui

local frame = Instance.new("Frame")
frame.Size = UDim2.new(0, 200, 0, 100)
frame.Position = UDim2.new(0.5, -100, 0.5, -50)
frame.BackgroundColor3 = Color3.fromRGB(50, 50, 50)
frame.Parent = screenGui

local button = Instance.new("TextButton")
button.Size = UDim2.new(1, 0, 1, 0)
button.Text = "Click Me!"
button.BackgroundColor3 = Color3.fromRGB(0, 162, 255)
button.TextColor3 = Color3.new(1,1,1)
button.Parent = frame

button.MouseButton1Click:Connect(function()
    button.Text = "Clicked!"
    wait(1)
    button.Text = "Click Me!"
end)

Adding Polish

Hovers? Use MouseEnter/MouseLeave. TweenService for smooth animations.

local TweenService = game:GetService("TweenService")

local hoverTween = TweenInfo.new(0.2, Enum.EasingStyle.Quad, Enum.EasingDirection.Out)
local hoverProps = {BackgroundColor3 = Color3.fromRGB(0, 120, 200)}

button.MouseEnter:Connect(function()
    TweenService:Create(button, hoverTween, hoverProps):Play()
end)

button.MouseLeave:Connect(function()
    local leaveProps = {BackgroundColor3 = Color3.fromRGB(0, 162, 255)}
    TweenService:Create(button, hoverTween, leaveProps):Play()
end)

Pro Moves

  • UDim2 for responsive sizing—scale with screen.
  • Connect to remotes for server actions.
  • Test on mobile; buttons gotta be thumb-friendly.

Quick Tip

UIListLayout keeps things tidy. No manual positioning headaches.
